# Project Memory: one-extension-to-pycharm

## Project Overview
- **Name**: 一键配置 PyCharm 风格 (One-Click PyCharm Style)
- **Type**: VS Code Extension (also supports Cursor)
- **Purpose**: Configures VS Code/Cursor with PyCharm-like development environment and provides productivity tools
- **Tested on**: Ubuntu 20.04, 22.04
- **Repository**: https://github.com/HuiyangCao/vscode-copy-with-ref.git

## Tech Stack
- **Language**: TypeScript (strict mode)
- **Runtime**: Node.js + npm
- **Framework**: VS Code Extension API (^1.80.0)
- **Build**: TypeScript compiler (tsc) with custom tsconfig in other_files/
- **Package**: @vscode/vsce for .vsix packaging
- **Target**: ES2020, CommonJS modules

## Key Directories
- `src/` - TypeScript source code
- `out/` - Compiled JavaScript output (gitignored)
- `other_files/` - Config files, assets, scripts (tsconfig.json, gen_icon.py, uninstall_config.py, template_commands.json, icon.png)
- `contributes/` - Separated package.json contributes (auto-merged by scripts/mergeContributes.js)
- `scripts/` - Build scripts (mergeContributes.js)
- `doc/` - Documentation files
- `JetBrainsMono-2.304/` - Bundled font files
- `node_modules/` - Dependencies (gitignored)

## Source Code Architecture

### Core Files
| File | Purpose |
|------|---------|
| `src/extension.ts` | Entry point - activate/deactivate, loads config, registers all modules |
| `src/constants.ts` | Global EXTENSION_ID = 'user_extension' |
| `src/config.ts` | Load config.json, apply VS Code settings and keybindings |

| `src/webExplorer.ts` | Web Explorer TreeView - bookmark manager for URLs |

### Configuration
- `config.json` - Settings and keybindings applied on activation
- `~/.config/user_extension/command_config/*.json` - User command configurations
- `~/.config/user_extension/bookmarks.json` - User bookmarks

## Build & Install Commands
- **Compile**: `npm run compile` (uses other_files/tsconfig.json)
- **Watch**: `npm run watch`
- **Package**: `vsce package --no-dependencies`
- **Install**: `./install.sh` (auto-detects VS Code and Cursor, installs dependencies, compiles, packages, installs)
- **Uninstall**: `./uninstall.sh` (reverts settings, uninstalls extension, optional cleanup)
- **Reinstall**: `./reinstall.sh`

## Features

### 1. PyCharm Style Configuration
- Theme: JetBrains Darcula Theme
- Font: JetBrains Mono, 14px with ligatures
- Applies to: editor, terminal, debug console, notebook, chat, markdown, GitLens

### 2. File Tree Optimization
- Double-click to expand folders
- Double-click to open files
- Pinned tabs on separate row

### 3. Clipboard Operations
- **Copy with File Reference** (Ctrl+Shift+C): `@filepath:line` or `@filepath:start-end`
- **Copy File Name**: Right-click in explorer
- **Copy to System Clipboard**: GNOME file manager format (requires xclip)

### 4. Quick Folder Navigation
- Ctrl+Alt+E: Folder search picker with favorites support

### 5. Debug Enhancements
- One-click kill all Python debug processes
- Resolves ${input:xxx} in launch.json with last-value memory

### 6. Command Manager
- TreeView in activity bar
- JSON-based command configuration from `~/.config/user_extension/command_config/*.json`
- **Auto parameter scanning**: extracts `{paramName}` from command strings automatically, no `parameter_refs` needed
- **Parameter types**:
  - `select` - QuickPick dropdown
  - `string` - InputBox text
  - `auto` - Executes command in target terminal via `export __AUTO_xxx=$(cmd)`, displays value, then uses `$VAR` in final command
- **Escape syntax**: `\{literal\}` preserves literal braces in command strings
- File watchers for hot reload
- Edit Config button inline on category items

### 7. SSH Manager
- Parses ~/.ssh/config
- Quick connect to SSH hosts
- Auto-refresh on config changes

### 8. Web Explorer (Bookmarks)
- Add/remove URL bookmarks
- Opens in Simple Browser or external browser
- Stored in ~/.config/user_extension/bookmarks.json

## State Management
- **workspaceState**: Favorite folders, command parameters, debug inputs
- **globalState**: jetbrainsNotified flag
- **File-based**: bookmarks.json, command_config/*.json

## Command Config Format
```json
{
  "parameters": {
    "param_name": {
      "type": "select" | "string" | "auto",
      "prompt": "Description",
      "options": ["opt1", "opt2"],  // for select
      "command": "shell command"    // for auto: executed in terminal to get value
    }
  },
  "commands": [
    {
      "name": "Display Name",
      "command": "shell command with {param_name} placeholders, \\{literal\\} for braces"
    }
  ]
}
```
- `parameter_refs` removed — parameters are auto-detected from `{...}` in command strings
- `auto` type: exports as env var `__AUTO_xxx=$(command)`, echoes value, then uses `$__AUTO_xxx` in final command
- `\{...\}` escapes literal braces

## Important Notes
1. **EXTENSION_ID** in constants.ts is the single source of truth for command/view IDs
2. Config application is additive - only adds/updates, never deletes user configs
3. Uninstall uses Python script to cleanly revert settings.json and keybindings.json
4. Command config files starting with `_` are ignored
5. File watchers auto-refresh command tree and SSH list on config changes
6. Debug input values persist across sessions in workspaceState
7. SSH manager only parses Host, Hostname, User, Port fields
8. Web explorer uses Simple Browser (VSCode 1.76+) with external browser fallback

## Known Limitations
